Getting Started with Python

By Cooper Hollmaier | May 6, 2020, 11:49am PST

Whether you're a software engineer, data scientist, or just someone who wants to make tediuous tasks easy, Python might be the programming language you've been dreaming of. I recently started investigating the language and I have to say that I'm impressed. It's remarkably easy to get rolling (especially if you're on a Mac) and I was amazed by all the things I could use Python to do.

In the article to follow, we'll walk through setting up Python on your machine, the problem I solved for my own workflow by building a program to parse HTML, and some helpful reading if you want to take your learning a step further.

Note: the program we are about to build can make it very easy to extract content from a webpage. It is important that we are "good citizens of the web" and parse only information that we own or have permission to access.

Now that we've got the ethics out of the way, let's help access Python and all the goodness it has to offer.

Installing Python

Before we start writing code, it is important to pick a good text editor. This tutorial will make use of the out-of-the-box file editor that Python gives us, but there's a ton of other great options such as PyCharm or Visual Studio Code

One of the reasons I have loved working in Python is because of how easy it is for someone who has never coded before to get set-up for the first time.

If you're on a Mac:
Browse to the Mac OS X downloads section and find the most recent stable release (at the time of this writing, we're on version 3.7.9). You'll want the "macOS 64-bit installer".

If you're on a PC:
Browse to the Windows downloads section and find the latest stable release of Python. You'll want the "x86 Executable Installer".

If you're running some different operating system:
Don't worry. There's a downloads section for you too.

The Moving Parts

The software you've just installed on your computer is what's called an Integrated Development Environment or IDE. This is a set of tools that make a developer's job easier. In our case, the Python IDE comes with 2 different tools - an interactive shell and a file editor. We'll be using both to build our HTML parser.

The Integrated Development Environment

The Interactive Shell

Think of this kind of like a calculator. You can punch stuff in on-demand and get back a response quickly. The interactive shell will allow you to run code that you write, 1 item at a time. This shell is also where you'll be able to see the outputs from the program that we'll be building.

The File Editor

The file editor contains all the code that you'll need to run your complete program. It will include a detailed list of instructions that you'll be giving to the computer to run with but it's not interactive like the interactive shell. Anything you write in the file editor must be saved and then run.

The Key Principles

Dictionaries

These work similar in Python to how they did in your highschool english class. A dictionary in Python is a data structure that contains an unordered list of values. We can access the data in this dictionary by referring to it's "key". In the program we're building, we'll build a dictionary to select specific elements in the HTML. I will admit, this wasn't my first approach to the code, but I went back and found that using a dictionary drastically cut down on the code I had to repeat.

Else if statements

Abbreviated as elif in our code, these statements allow us to test if a condition is true or false when there are multiple potential cases possible. We'll use these heavily in our example to follow.

If statements

We use if statements to decide whether to run a block of code or skip over it.

Import()

Sometimes someone has already developed an elegant solution to a problem so we don't have to. We use the import statement at the begining of a program to add additional functionality that we want to use later in the program.

Input()

This function prompts the user for keyword input.

Len()

This function returns the number of items in the object it's called on.

Libraries

Libraries are common in any programming language. These are reusable snippets of code that we'll use to perform complex actions. Other developers have built out useful functionality that is has many uses and can be applied to different problems.

Lower()

This method returns a lowercased string

Print()

This method is used for printing data or a message to the screen. When we use this method in our program, it'll display our text in the interactive shell.

Str()

This method returns a string of the contained object.
